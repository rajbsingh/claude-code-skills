# MVP Scope Builder Skill

## Purpose

Help define a realistic MVP that can be built, tested, and validated without wasting time or money.

## Use this skill when

- The user has an app or SaaS idea
- The user wants to build an MVP
- The user is confused about features
- The user wants to avoid overbuilding
- The user wants investor-ready product clarity
- The user needs a first version roadmap

## Instructions

1. Identify the target user.
2. Define the core pain point.
3. Separate features into:
   - Must-have for MVP
   - Nice-to-have later
   - Avoid for now
4. Suggest the simplest testable version.
5. Identify risks:
   - technical risk
   - market risk
   - legal/compliance risk
   - cost risk
   - user adoption risk
6. Suggest a validation plan before full development.
7. Give a practical build roadmap.

## Output format

Use this structure:

## MVP Goal
Explain the MVP goal in simple language.

## Target Users
List the primary users.

## Must-Have Features
List only the essential features.

## Later Features
List features to add after validation.

## Avoid for Now
List features that should not be built yet.

## Validation Plan
Explain how to test demand.

## Build Roadmap
Give a simple phase-wise roadmap.
